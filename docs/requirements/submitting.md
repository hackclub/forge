# Submitting your design

So, you've designed a beautiful project digitally, and now its time to get the money to make it real 🤑


Before approving your project, we need to make sure that the project meets our ✨ Submission Requirements ✨

We want to see real, polished and [shipped](/docs/requirements/shipping) projects.


Generally speaking, each project must be closer to a product than a demo - that doesn't mean go ultra advertising mode, but that does mean that a breadboarded together project with no case *doesn't* count.


## You should have:
- A public Github (or other version tracking platform) repository containing:
    - A good README
    - Source files for your project
        - This includes hardware design, firmware, 3D assemblies, etc
    - Production files (if applicable)
    - JOURNAL.md if journalling on Git
    - BOM.csv complete with functioning links (where applicable)
- A cover image, title, and description on Forge
- Your repository linked to Forge

## You should not have: 
- AI Generated READMEs or Journal entries
- Stolen work from other people
- Missing firmware/software

> NOTE: Your project will be rejected if any of these are missing, and resubmitting will put you at the back of the queue again. Please double and triple check these before submitting!

## README.md

Your README should contain: 
- A short description of what your project is
- A couple sentences on why you made the project
- PICTURES OF YOUR PROJECT
- A screenshot of a full 3D model with your project
- A screenshot of your PCB, if you have one
- A screenshot of your schematic, if you have one
- A wiring diagram, if you're doing any wiring that isn't on a PCB
- A BOM in table format at the end of the README
